# 全来店报价单模板需求说明（提给 `quanlaidian-quote-service`）

> **适用范围**：本文档既适用于新建报价单模板，也适用于对现有模板的差异化改造。
>
> **来源**：参考客户在 [quanlaidian-quote-skills #4](https://github.com/jasonshao/quanlaidian-quote-skills/issues/4) 提供的 `盼盼食品-全来店报价单V2.xlsx`（100 套锚点版）。
>
> **读者**：`quanlaidian-quote-service` 后端 / 模板工程同学。
>
> **交付物**：本期只改报价单 PDF / Excel 模板和渲染逻辑，不改 API 协议、不改 skill 表单字段。

## 1. 背景与目的

- skill `quanlaidian-quotation-skill` 负责收集业务字段并调用本服务出报价。
- 客户希望报价单在视觉和内容上更接近 `盼盼食品-全来店报价单V2.xlsx`，具体要求：
  1. 抬头固定放 **客户品牌 logo + 收钱吧 logo + 全来店 logo** 三联。
  2. 每个套餐 / 模块行后面的 **K 列『套餐说明 / 其他说明』** 要带上对应文案。
  3. 底部追加 **『权益类』** 附加条款区块（小程序验证次数、外卖接单费用）。
- 文字内容已统一维护在 skill 仓库的 [`references/product_catalog.md`](https://github.com/jasonshao/quanlaidian-quote-skills/blob/main/references/product_catalog.md)，**文本单一权威来源为该文件**。模板实现时：
  - 第一期可先**硬编码**权益类文本（两段）与 SKU→套餐说明的映射。
  - 后续如需动态化，通过 `quanlaidian-quote-service` 从 catalog 源文件读取，不由 skill 侧下发。

## 3. 模板布局规范

### 3.1 抬头区（A1:K3 合并）

三联 logo 横向排布：

| 位置 | 内容 | 来源 |
|---|---|---|
| 左（约 1/3 宽） | **客户品牌 logo** | 客户提供时渲染客户 logo；未提供时渲染品牌文字占位（沿用现有品牌名即可） |
| 中（约 1/3 宽） | **收钱吧 logo** | 由模板侧维护的固定资产 |
| 右（约 1/3 宽） | **全来店 logo** | 由模板侧维护的固定资产 |

布局建议：

- 三 logo 底部对齐；垂直居中于抬头行带；三段之间留等宽留白
- 抬头行带整体高度约 3 行 Excel 行高（样例 xlsx 的 A1:K3 合并高度作参考）
- PDF 导出时 logo 清晰度 ≥ 200 dpi，避免拉伸变形
- Logo 资产文件名建议：`assets/logo_shouqianba.png`、`assets/logo_quanlaidian.png`；客户品牌 logo 走请求侧上传或品牌库映射

### 3.2 主表列结构

按样例 xlsx 对齐：

| 列 | 标题 | 说明 |
|---|---|---|
| A | 商品分类 | 示例：`套餐` / `供应链管理` / `全链通` / `单次实施与售后服务` |
| B | 商品名称 | 套餐或模块名（含"包含模块"子项） |
| C | 单位 | `店/年` / `个/年` / `号/年` / `人天` 等 |
| D | 1 套单价 | 标准单价（标价） |
| E / F | 锚点 1 优惠单价 / 总价 | 对应请求门店数档位 1 |
| G / H | 锚点 2 优惠单价 / 总价 | 对应请求门店数档位 2 |
| I / J | 锚点 3 优惠单价 / 总价 | 对应请求门店数档位 3 |
| K | 套餐说明 / 其他说明 | 按 SKU/模块查 catalog 填入（详见 §3.3） |

**阶梯列随请求门店数动态**（对齐 skill 现有 31–300 段锚点规则）：

- `门店数 N < 30`：主列展示 N 店实际单价 + 总价；相邻两档对比可留空或隐藏
- `30 ≤ N < 50`：展示 `[30, 50]` 两档
- `50 ≤ N < 100`：展示 `[50, 100]`
- `100 ≤ N < 200`：展示 `[100, 200]`
- `200 ≤ N ≤ 300`：展示 `[200, 300]`
- `N ≥ 301`：不生成，服务端返回 422，由 skill 转人工
- 样例 xlsx 为 100 套版，展示 `1 / 30 / 50 / 100` 四档；新模板按当次请求动态定档

**合并与总价行**：

- 同一大类下子项合并 A 列（如 xlsx `A4:K4`、`A12:A14`）
- 每档总价在"总计价格"行汇总（样例 xlsx A17 行为 `总计价格`，E/G/I 列为对应档位标签，F/H/J 列为金额）

### 3.3 K 列『套餐说明』数据源

数据源：skill 仓库 `references/product_catalog.md` 的以下内容。

**套餐行**（SKU→文本 查 `## 一 / 二` 下的 `### 1.1 套餐说明` 小节）：

示例 QC-04 `轻餐连锁营销旗舰版`（对应样例 xlsx K6 原文）：

```
① 商户中心：组织结构 + 基本档管理 + 参数管理 + 报表
② 点餐收银：先付后吃模式，收银机上套餐点餐、团购核销、自助称重、单品点餐、打印账单、外卖接单等
③ 商家独立小程序：提供线上扫码点餐、支付、交易场景，小程序属于商家自己，需要商家在微信开通后对接
④ 会员营销：企业会员批量办理与充值、会员储值、会员消费、会员查询、数据统计；会员类别、会员规则、会员储值活动、会员群体、会员库、会员报表、会员营销活动、会员礼品设置、各门店会员交易查询等
⑤ 门店库存：门店库存管理是单店管理模式下的库存管理系统，涵盖门店独立采购管理（采购、验货、退货）等，以及门店间调拨
```

其他 SKU（QC-01/02/03/05、ZC-01~05）的套餐说明见 catalog 文件，结构一致。

**模块行**（模块名 → 『说明』列文本，查 `## 一 / 二` 下的 `### 2 门店增值模块` / `### 3 总部模块` 表格）：

- 样例 xlsx K12 成本管理 原文已反向写入 catalog 的 `成本管理` 说明行
- 样例 xlsx K14 配送中心 / 生产加工 原文已反向写入 catalog 的 `配送中心` / `生产加工` 说明行

**实施服务行**：样例 xlsx K16 原文已写入 catalog `三、权益类` 的『实施服务人天测算口径』小节：

```
按照具体项目测算：10 家门店以下 5 人天、11–50 家门店 10 人天、51–300 门店 20 人天、300 以上门店 30 人天；1 个生产加工模块另计 10 人天。
```

模板渲染时，K 列按 row 类型（套餐 / 模块 / 实施服务）查对应数据源，**空值不换占位符文本**，留空即可。

### 3.4 权益类页脚

按样例 xlsx 第 18 行：`A18` 单元格 "权益类"；`B18:K18` 合并，放完整正文。

**完整正文（建议直接嵌入模板；如后续动态化，从 `product_catalog.md` §三『权益类』读取）**：

```
【小程序手机验证次数】
* 平台小程序开通后需要按照用量在商户中心后台自行充值手机验证次数，25 元 500 次，150 元 5000 次。
* 商家版小程序，需在微信公众平台进行手机验证次数权益包充值，具体费用见微信公众平台要求。

【外卖接单费用】
* 如品牌侧有外卖品牌总账户，申请美团、淘宝闪购外卖品牌商账户成功后，可免除连锁账号下的门店外卖接单费及接口费。
* 如门店未入驻外卖品牌总账户，三方外卖接单按照用量在商户中心后台充值：按单（只收正向单）收基础费 1 分钱每单，保留基础接口调用以保证商家正常接单。
* 增值接口费：
  - 骑手信息查询：0.0015 元 / 次。系统不再自动同步外卖平台配送单状态，门店若想联系骑手，可在 POS 端订单详情页手动触发查询。
  - 查询收货人真实地址（美团商家自配送）：0.003 元 / 次，建议美团商家自配送场景开启此开关。
  - 库存实时同步到外卖平台：0.003 元 / 次，商品库存变动时将库存数量实时同步到外卖平台。
  - 商家确认出餐：0.0015 元 / 次，默认自动开启。
```

排版建议：小标题加粗；星号列表缩进；『增值接口费』四项子项二级缩进。PDF 字号建议正文 9pt，小标题 10pt 加粗。

## 4. 数据流

```
OpenClaw 表单 / 引导式对话
        │
        ▼
  skill (quanlaidian-quote-skills)
   └─ 拼装业务 JSON + POST /v1/quote
        │
        ▼
  quanlaidian-quote-service (本项目)
   ├─ 读 catalog：价格 / 套餐说明 / 模块说明 / 权益类
   ├─ 渲染 PDF / Excel：抬头三 logo + 主表 + K 列 + 权益类页脚
   └─ 返回文件 URL（7 天有效）
        │
        ▼
  skill 侧拿到 URL → 在对话中展示预览 + 下载入口
```
