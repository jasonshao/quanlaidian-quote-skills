# 在飞书里用 OpenClaw 机器人出全来店报价

> 面向：销售 / 售前
> 5 分钟读完，看完就能出第一份报价

---

## 它能帮你做什么

你只要在飞书里**像聊天一样**告诉机器人：客户品牌、餐饮类型、门店数、套餐 —— 几句话之后机器人就会回你三份文件：

| 文件 | 用途 |
|---|---|
| **PDF** | 直接转发给客户 |
| **Excel** | 内部复核、改折扣、做二次调整 |
| **JSON** | 留档，下次用同一份配置可以一键再生成 |

不需要打开 Excel 模板、不需要查价格表、不需要算折扣 —— 机器人会按门店数自动给推荐折扣，按所选套餐和模块查标准价、算合计。

带着这几个信息再呼机器人，会一次过完成，不用反复返工：

| 必须有 | 怎么填 |
|---|---|
| 客户品牌名称 | 报价单抬头会用，写客户对外用的全称 |
| 餐饮类型 | `轻餐` 或 `正餐`（二选一） |
| 门店数量 | 1 – 300 店（≥ 301 店要转人工）。31–300 店自动落到 50/100/200/300 最近锚点，机器人会随报价附相邻两档对比表 |
| 门店套餐 | 默认走对应餐饮类型的 `旗舰版`，客户没说就按默认 |

| 按需有 | 什么时候要 |
|---|---|
| 门店增值模块 | 客户额外要 KDS、成本管理、电子发票等才填 |
| 总部模块 | 客户有总部需求（配送中心、生产加工、企微 SCRM）才填 |
| 配送中心 / 生产加工中心数量 | 选了对应总部模块时，机器人会再追问 |
| 成交价系数 | 标准报价直接走默认；**只有要人工改价**才填 |
| 人工改价原因 | **填了成交价系数就必须填原因**（如"战略客户首单特批"） |
| 实施服务 | 客户要顾问服务 / 定制开发才填 |

## 第一步：呼出技能

两种方式都行：

**A. 群里 @机器人**

```
@<!-- TODO: 机器人名 --> 帮我出一份全来店报价
```

<!-- TODO: 如果机器人有规定的触发关键字（如必须包含"全来店"或"报价"），在这里写明 -->

**B. 私聊机器人**

直接打开和机器人的私聊窗口，发：

```
帮我给 XX 品牌出一份全来店报价
```

机器人识别到「全来店报价」相关意图后，会**优先弹出原生表单**让你填；如果没弹表单，就直接进入对话式问答（见下文「没弹出表单怎么办」）。

## 第二步（推荐）：填表

机器人弹出的表单大致长这样：

<!-- TODO: 在这里贴一张原生表单截图 -->

填表要点：

- **必填四件套**先填好：品牌名、餐饮类型、门店数、套餐
- **增值模块 / 总部模块**：勾选你确认要的，没把握的不勾（事后可以再出一版）
- **勾了「配送中心」**会出现「配送中心数量」字段，默认 1
- **勾了「生产加工」**会出现「生产加工中心数量」字段，默认 1
- **想改价**：填「成交价系数」（例如 `0.25` 表示按标准价 25% 成交），同时**必须**写「人工改价原因」，不写会失败
- 不需要的字段就空着，不要瞎填

填完点提交，跳到「第三步」。

## 第二步备选：没弹出表单怎么办？

不要慌，机器人会自动切到**对话模式**，按顺序一条一条问你。你可以直接回中文名或数字，例如：

```
机器人：客户品牌叫什么？
你：黑马汇

机器人：餐饮类型？轻餐 还是 正餐？
你：轻餐

机器人：门店数量？
你：20

机器人：用哪个套餐？默认推荐"轻餐连锁营销旗舰版"
你：用基础版

机器人：要加哪些门店增值模块？回复"无"或直接说名字（可多选）
你：厨房KDS

机器人：总部模块呢？(配送中心 / 生产加工 / 企业微信SCRM / 商家小程序号)
你：无

机器人：好的，按标准价生成。要不要改价？
你：不用
```

机器人收齐信息后会先回一段**配置摘要**让你确认，再去生成文件。

## 第三步：拿文件

机器人成功生成后，会**一次性**回给你两块内容：

```markdown
## 本次配置摘要

- 品牌：黑马汇
- 餐饮类型：轻餐    门店数：20
- 套餐：轻餐连锁营销基础版
- 总价：¥71,820

## 下载文件

- [报价单 PDF](...)
- [报价单 Excel](...)
- [报价配置 JSON](...)
```

**做两件事：**

1. **核对摘要** — 品牌名、门店数、套餐、总价对不对？不对就直接告诉机器人「重新出，门店数改成 30」之类，不用从头再来
2. **及时保存文件** — 三个链接 **7 天有效**，过期就要重新生成。建议立刻把 PDF 转给客户，把 JSON 存到自己的客户文件夹

## 常见情况

| 你看到 | 是什么意思 / 怎么办 |
|---|---|
| 机器人提示「折扣超出建议范围」 | 你填的成交价系数偏低，机器人**会照常生成**，但需要你确认是否走特批流程 |
| 机器人说「这个规模建议转人工」 | 门店数 ≥ 301，或选了非标准联购，按内部流程找售前 |
| 机器人回「服务端错误 XXX」 | 后端临时问题，**截图聊天 + 把客户信息发给 IT 对接人**（见下方「找谁帮忙」） |
| 机器人回「网络异常」 | 后端连不上，等 1 分钟重试一次；连续失败 → 报 IT |
| 机器人回「人工改价必须填写原因」 | 你填了「成交价系数」但没填「人工改价原因」，把原因补上重新提交 |
| 文件链接打不开 | 检查是不是超过 7 天了；过期就回机器人「重新出一份」 |

## 附录：进阶用法

### 1. 复用上次报价

把上次机器人给你的 **JSON 文件**重新发给机器人，说「按这份配置再出一份」，可以避免重新填一遍。适用场景：

- 同一个客户同一份配置，只是 PDF 链接过期了
- 需要给客户保留同一报价版本的可追溯记录

### 2. 给同一客户做对比报价

客户还没定门店数？分别用 10 店 / 20 店 / 30 店各出一份，PDF 并排发给客户做对比。大客户可以用 50 / 100 / 200（或 100 / 200 / 300）三档对比 —— 直接告诉机器人「出 50/100/200 三档对比」，它会串联调用三次报价，给你三组文件 + 一张并排对比摘要。

或者用「是否启用阶梯报价」字段一次出多个规模的对比表（在表单里勾上即可）。该开关只在 ≤30 店段生效；31 店以上 API 始终自动附带相邻两档对比，不需要再勾。

### 3. 战略客户改价

要给老客户、KA 客户专门改价时：

1. 填「成交价系数」（例如 `0.85` = 八五折）
2. **必须**填「人工改价原因」，写清楚审批依据，例如「年度续约优惠，已获区域销售总监口头特批」
3. 这条原因会写入 JSON 留档，将来审计可查
